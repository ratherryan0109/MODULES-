Structured Data Examples

Open your browser's developer console to inspect the JSON-LD blocks after the page loads.

Example 1: Article Schema (JSON-LD)

Article: Understanding CSS Grid

By Sarah Johnson | Published: June 10, 2026

CSS Grid is a powerful layout system that allows for two-dimensional layouts...

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Understanding CSS Grid Layout",
  "author": {
    "@type": "Person",
    "name": "Sarah Johnson"
  },
  "datePublished": "2026-06-10",
  "dateModified": "2026-06-12",
  "description": "A comprehensive guide to CSS Grid Layout",
  "image": "https://example.com/css-grid.jpg",
  "publisher": {
    "@type": "Organization",
    "name": "WebDev Daily"
  }
}
</script>
Explanation: This JSON-LD block describes an article with its author, publication date, and publisher. Google can use this to generate a rich snippet in search results showing the author name and publication date.

Example 2: Product Schema

Product Image

Premium Wireless Headphones

Noise-cancelling Bluetooth 5.0 headphones with 30-hour battery life

Brand: AudioPro

Price: $79.99 In Stock

Rating: ⭐⭐⭐⭐½ (234 reviews)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Premium Wireless Headphones",
  "description": "Noise-cancelling Bluetooth 5.0 headphones",
  "sku": "APH-2026-001",
  "brand": {
    "@type": "Brand",
    "name": "AudioPro"
  },
  "offers": {
    "@type": "Offer",
    "price": "79.99",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.5",
    "reviewCount": "234"
  }
}
</script>
Explanation: The Product schema includes pricing, availability, brand, and ratings. Search results can display a price tag, stock status, and star rating alongside the product link.

Example 3: FAQ Schema

Frequently Asked Questions

What is structured data?

Structured data is a standardized format for providing information about a webpage.

Why use JSON-LD?

JSON-LD is Google's recommended format because it separates data from HTML.

How do I test structured data?

Use Google's Rich Results Test or Schema.org Validator.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is structured data?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Structured data is a standardized format for providing information about a webpage."
      }
    },
    {
      "@type": "Question",
      "name": "Why use JSON-LD?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "JSON-LD is Google's recommended format because it separates data from HTML."
      }
    },
    {
      "@type": "Question",
      "name": "How do I test structured data?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Use Google's Rich Results Test or Schema.org Validator."
      }
    }
  ]
}
</script>
Explanation: FAQ schema enables your questions and answers to appear directly in search results as an expandable FAQ snippet, increasing visibility and click-through rate.

Example 4: Event Schema

Web Developer Conference 2026

📅 September 15-17, 2026 | 9:00 AM - 6:00 PM

📍 Convention Center, 123 Main Street, San Francisco, CA

🎫 Tickets: $299 | Buy Tickets

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Event",
  "name": "Web Developer Conference 2026",
  "startDate": "2026-09-15T09:00:00",
  "endDate": "2026-09-17T18:00:00",
  "location": {
    "@type": "Place",
    "name": "Convention Center",
    "address": {
      "@type": "PostalAddress",
      "streetAddress": "123 Main Street",
      "addressLocality": "San Francisco",
      "addressRegion": "CA",
      "postalCode": "94102"
    }
  },
  "offers": {
    "@type": "Offer",
    "url": "https://example.com/tickets",
    "price": "299",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  }
}
</script>
Explanation: Event schema allows search engines to display event dates, location, and ticket prices directly in search results. Users can add the event to their calendar with one click.

Example 8: Video Schema

▶ Video Thumbnail

HTML for Beginners - Full Course

Duration: 30:15 | Uploaded: January 10, 2026

{
  "@context": "https://schema.org",

  "description": "Learn HTML from scratch with this comprehensive tutorial",
  "thumbnailUrl": "https://example.com/thumbnail.jpg",

Explanation: Video schema helps videos appear in video search results with thumbnail, duration, and upload date. It can also enable rich snippets with play buttons in search results.

Example 9: Multiple Schema Types

Important: You can include multiple JSON-LD blocks on a single page. Each block is wrapped in its own <script type="application/ld+json"> tag. 
<!-- WebPage schema -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "About Us",
  "description": "Learn about our company",
  "breadcrumb": {
    "@type": "BreadcrumbList",
    "itemListElement": [
      {
        "@type": "ListItem",
        "position": 1,
        "name": "Home",
        "item": "https://example.com"
      },
      {
        "@type": "ListItem",
        "position": 2,
        "name": "About",
        "item": "https://example.com/about"
      }
    ]
  }
}
</script>

<!-- Organization schema -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "TechCorp",
  "url": "https://techcorp.example.com"
}
</script>

<!-- Person schema for CEO -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Jane Smith",
  "jobTitle": "CEO",
  "worksFor": {
    "@type": "Organization",
    "name": "TechCorp"
  }
}
</script>
Key Point: Each JSON-LD block is independent. You can combine as many schema types as needed to fully describe your page content.

Example 10: Local Business Schema

The Gourmet Kitchen

🍝 Italian Restaurant | Price: $$

📍 456 Oak Avenue, New York, NY 10001

⭐ 4.3 (187 reviews)

🕐 Mon-Sat: 11:00 AM - 10:00 PM

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Restaurant",
  "name": "The Gourmet Kitchen",
  "servesCuisine": "Italian",
  "priceRange": "$$",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "456 Oak Avenue",
    "addressLocality": "New York",
    "addressRegion": "NY",
    "postalCode": "10001"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.3",
    "reviewCount": "187"
  }
}
</script>
Explanation: Local Business schema (including subtypes like Restaurant) helps local businesses appear in local search results, map packs, and knowledge panels with reviews, cuisine type, and price range.

Summary

These examples demonstrate the most common Schema.org types used in web development. To validate your structured data:

  1. Use Google Rich Results Test
  2. Use Schema.org Validator
  3. Check Google Search Console for errors

Remember: Structured data must always match the visible content on the page.
