Example 1: Semantic vs Non-Semantic Layout

❌ Non-Semantic (Div Soup):
My Website
Home
About
Article Title
Content here...
© 2026
✅ Semantic HTML:

My Website

Home | About

Article Title

Content here...

© 2026

<!-- Non-semantic -->
<div class="header">...</div>

<!-- Semantic (preferred) -->
<header>
  <h1>My Website</h1>
  <nav>
    <a href="#">Home</a>
  </nav>
</header>
<main>
  <article>
    <h2>Article Title</h2>
  </article>
</main>
<footer>...</footer>
Why it matters: Semantic elements improve accessibility (screen readers understand the structure), SEO (search engines identify content areas), and code readability.

Example 2: Image Alt Text Best Practices

❌ Bad - Missing or poor alt text:

<img src="chart.jpg"> - No alt text at all

✅ Good - Descriptive alt text:

<img src="chart.jpg" alt="Bar chart showing 40% increase in Q1 2026 revenue">

✅ Decorative image - empty alt:

<img src="decoration.png" alt="" role="presentation">

<!-- Informative image -->
<img src="chart.png" alt="Bar chart showing 40% increase in Q1 2026 revenue">

<!-- Decorative image -->
<img src="border.png" alt="" role="presentation">

<!-- Linked logo -->
<a href="/">
  <img src="logo.png" alt="Company Name - Home">
</a>

Example 3: Accessible Form Implementation

❌ Bad Form:

Missing labels, relies on placeholders only

✅ Good Form (Accessible):
Enter your first and last name
We will never share your email
Contact preference 
<!-- Each input needs a label -->
<label for="full-name">Full Name:</label>
<input type="text" id="full-name" name="full-name" required>

<!-- Fieldset groups related inputs -->
<fieldset>
  <legend>Contact preference</legend>
  <input type="radio" id="pref-email" name="preference">
  <label for="pref-email">Email</label>
</fieldset>

Example 4: Link Best Practices

❌ Bad Links:
Click here for more information.
External Link
✅ Good Links:
Read our complete guide to semantic HTML
External Resource (opens in new tab)
Download PDF (2.4 MB) 
<!-- Descriptive link text -->
<a href="/guide">Read our complete guide to semantic HTML</a>

<!-- External link safety -->
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
  External Resource
</a>

<!-- Skip link for keyboard users -->
<a href="#main-content" class="skip-link">Skip to main content</a>

Example 5: Complete Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title - Site Name</title>
  <meta name="description" content="Brief page description">
  <meta name="author" content="Author Name">
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header>
    <nav aria-label="Primary">
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <article>
      <h1>Main Title</h1>
      <section>
        <h2>Section Title</h2>
        <p>Content...</p>
      </section>
    </article>
    
    <aside>
      <h2>Related Content</h2>
    </aside>
  </main>
  
  <footer>
    <p>&copy; 2026 Company Name</p>
  </footer>
  
  <script src="scripts.js" defer></script>
</body>
</html>
Structure rules: One <h1> per page, <main> is unique, <nav> for navigation, <aside> for tangentially related content. Always validate with W3C validator.

Example 6: Performance Best Practices

❌ Performance Issues:

JavaScript in <head> blocks rendering

No lazy loading on images

Missing dimension attributes on images

✅ Performance Optimized: 
<!-- Optimized image loading -->
<img src="photo.jpg" 
     alt="Description" 
     width="800" 
     height="600" 
     loading="lazy" 
     decoding="async">

<!-- Deferred JavaScript -->
<script src="app.js" defer></script>

<!-- Async third-party script -->
<script src="analytics.js" async></script>

Example 7: Proper Heading Hierarchy

❌ Bad Hierarchy (skipping levels):

Page Title

Sub-section

Detail
✅ Proper Hierarchy:

Page Title

Major Section

Sub-section

Detail

Another Major Section

<!-- Correct heading outline -->
h1 Page Title
├── h2 Major Section
│   ├── h3 Sub-section
│   │   └── h4 Detail
│   └── h3 Another sub-section
└── h2 Another Major Section

Example 8: Essential Meta Tags

<!-- Character encoding -->
<meta charset="UTF-8">

<!-- Viewport for responsive design -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- Page title (essential for SEO) -->
<title>Descriptive Page Title | Site Name</title>

<!-- Meta description (appears in search results) -->
<meta name="description" content="A concise description of this page's content, ideally 150-160 characters.">

<!-- Open Graph (social sharing) -->
<meta property="og:title" content="Page Title">
<meta property="og:description" content="Description for social sharing">
<meta property="og:image" content="https://example.com/social-image.jpg">
<meta property="og:url" content="https://example.com/page">
<meta name="twitter:card" content="summary_large_image">

<!-- Favicon -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
