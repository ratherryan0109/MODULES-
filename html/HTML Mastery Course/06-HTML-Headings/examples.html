Module 6: HTML Headings — Examples

Example 1: All Six Heading Levels

<h1>Heading Level 1</h1>
<h2>Heading Level 2</h2>
<h3>Heading Level 3</h3>
<h4>Heading Level 4</h4>
<h5>Heading Level 5</h5>
<h6>Heading Level 6</h6>

Example 2: Proper Heading Hierarchy

<h1>Web Development Guide</h1>
    <h2>Frontend Technologies</h2>
        <h3>HTML</h3>
        <h3>CSS</h3>
        <h3>JavaScript</h3>
    <h2>Backend Technologies</h2>
        <h3>Node.js</h3>
            <h4>Express Framework</h4>
        <h3>Python</h3>
            <h4>Django Framework</h4>

Example 3: Incorrect Hierarchy (Skipping Levels)

This creates confusion — avoid skipping heading levels.

<h1>My Page</h1>
    <h3>Section (skipping h2)</h3>  &larr; BAD
    <h4>Sub-section (skipping h3)</h4>  &larr; BAD
Outline (broken):

My Page

Section (skipping h2)

Sub-section (skipping h3)

Example 4: Styled Headings

Headings can be styled with CSS while maintaining semantic meaning.

<h1 style="color: #e74c3c; text-transform: uppercase;">Styled H1</h1>
<h2 style="border-bottom: 2px solid #333; padding-bottom: 5px;">Styled H2</h2>
<h3 style="font-style: italic;">Styled H3</h3>
Rendered Output:

Styled H1

Styled H2

Styled H3

Example 5: How Screen Readers See Headings

Screen reader users can navigate by heading to get a page overview.

Page Outline:
├── H1: HTML Mastery Course
│   ├── H2: Introduction
│   │   └── H3: What is HTML?
│   ├── H2: Core Concepts
│   │   ├── H3: Tags

│   └── H2: Summary
💡 Tip: Use the W3C HTML Validator or browser extensions that generate a heading outline to verify your heading structure.
