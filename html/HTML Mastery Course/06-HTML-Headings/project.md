# Mini Project: Build a 'HTML Headings Hierarchy' Demonstrator

## Objective
Create an educational HTML page that demonstrates proper heading hierarchy and its importance for SEO and accessibility.

## Requirements

Build a single HTML page with:

1. **Header**
   - Title: "HTML Headings Hierarchy Guide"
   - Navigation with links to sections

2. **Six Levels Demo Section**
   - Display each heading level from `<h1>` to `<h6>`
   - Next to each, show its default font size and a description of when to use it

3. **Proper Hierarchy Example Section**
   - Create a full article outline using proper heading structure
   - Topic: "How to Build a Website"

4. **Incorrect Hierarchy Example Section**
   - Show what NOT to do (skipping levels)
   - Explain why it is bad for accessibility

5. **Visual Outline Section**
   - Create a nested list (using `<ol>`/`<ul>`) that represents the heading outline of a web page
   - Show how the outline creates a table of contents

6. **Quick Reference Table**
   - A table listing all 6 heading levels with: tag, default size, bold, example usage

7. **Tips Section**
   - Best practices list
   - SEO tips
   - Accessibility tips

8. **Footer** with copyright

## Step-by-Step Instructions

1. Create `headings-guide.html`
2. Write the HTML5 document skeleton
3. Build the heading demo showing all six levels
4. Create the proper hierarchy example article
5. Create the incorrect hierarchy (bad example)
6. Build the visual outline with nested lists
7. Add the reference table
8. Include tips and best practices
9. Validate the HTML

## Bonus Challenges
- Add a script that generates the outline automatically from the headings on the page
- Add interactive toggles to show/hide the hierarchy examples
- Create a color-coded system for different heading levels
