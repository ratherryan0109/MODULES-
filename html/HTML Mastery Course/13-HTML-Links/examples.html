🔗 HTML Links — Code Examples

Click the links to see how they work. Check the code panel for implementation details.

Example 1: Basic Link Structure

The simplest link uses an <a> tag with an href attribute.

Visit Google
<a href="https://www.google.com">Visit Google</a>

The text inside the <a> tags is the visible, clickable part of the link.

Example 2: Absolute vs Relative URLs

Absolute URL: MDN HTML Docs (opens in new tab)

Relative URL (same directory): This same page

Relative URL (root): Site Root

<a href="https://developer.mozilla.org/">Absolute URL</a> <a href="examples.html">Relative URL (same dir)</a> <a href="/">Relative URL (root)</a>
Tip: Use absolute URLs for external sites and relative URLs for pages within your own site.

Example 3: The target Attribute

_self: Opens in same tab (default)

_blank: Opens in new tab

<a href="https://example.com" target="_self">_self: Same tab</a> <a href="https://example.com" target="_blank" rel="noopener noreferrer">_blank: New tab</a>
⚠ Security: Always add rel="noopener noreferrer" when using target="_blank".

Example 4: Anchor Links — Navigation Within a Page

Go to Section 1

Go to Section 2

Go to Section 3

<a href="#section1">Go to Section 1</a> <h3 id="section1">Section 1</h3>

Section 1 — Welcome!

You arrived here via the anchor link. The page scrolled to this section because the href="#section1" matches the id="section1" on this heading.

Section 2 — More Content

This is another section you can jump to. Anchor links are great for long pages like FAQs, documentation, and table of contents.

Section 3 — End of Examples

You can also link to sections on other pages: <a href="other-page.html#section">.

<!-- Link to another page's section --> <a href="about.html#team">View Team Section</a> <!-- Back to top --> <a href="#">Back to Top</a>

Example 5: Email and Telephone Links

📧 Send us an email (info@example.com)

📞 Call us: (123) 456-7890

📧 Email with pre-filled subject and body

<a href="mailto:info@example.com">Send Email</a> <a href="tel:+1234567890">Call Us</a> <a href="mailto:support@example.com?subject=Help%20Request&body=Hi%2C%20I%20need%20help%20with..."> Email with pre-filled fields </a>
Note: mailto: will open the user's default email client. tel: works best on mobile devices and some desktop VoIP apps.

Example 6: Download Links

📄 Download sample.txt file

📄 Download with custom filename

<a href="sample.txt" download>Download</a> <a href="sample.txt" download="custom-name.txt">Download with custom name</a>
Note: The download attribute only works for same-origin URLs.

Example 9: rel Attribute Values

External link with noopener + noreferrer

Link with nofollow (for SEO)

Link marked as external

<a href="https://external-site.com" rel="noopener noreferrer" target="_blank">Safe external link</a> <a href="https://spammy-site.com" rel="nofollow">No SEO credit passed</a> <a href="https://example.com" rel="external">Marked as external</a>
Always use rel="noopener noreferrer" when opening links in new tabs with target="_blank".

Example 10: Links vs Buttons (When to Use Which)

Use a link (anchor tag) for navigation:

Go to Example

Use a button for actions:

<!-- Link — navigates somewhere --> <a href="https://example.com">Go to Example</a> <!-- Button — performs an action --> <button onclick="alert('Clicked!')">Click Me</button>
Rule of thumb: Links navigate (they take you somewhere). Buttons perform actions (submit forms, open modals, trigger JavaScript). Using them correctly improves accessibility.

Example 11: Link Accessibility — Good vs Bad

❌ Bad: Click here to read our documentation.

✅ Good: Read our documentation about web standards.

✅ Good (new tab): Documentation (opens in new tab)

<!-- BAD: "Click here" has no context --> <a href="...">Click here</a> <!-- GOOD: Link text describes the destination --> <a href="...">documentation about web standards</a>
