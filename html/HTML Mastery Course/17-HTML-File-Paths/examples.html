📁 HTML File Paths — Code Examples

Example 1: Project Directory Structure

Understanding the file structure helps with paths. Here's a typical project:

website/ ├── index.html ← You are here ├── css/ │ └── style.css ├── js/ │ └── script.js ├── images/ │ ├── logo.png │ └── gallery/ │ └── photo.jpg └── pages/ ├── about.html └── blog/ └── post.html

Example 3: Same Directory

<!-- Both files are in the same folder --> <a href="about.html">About Us</a> <img src="logo.png" alt="Logo">

Just use the filename directly.

Example 4: Subdirectory Navigation

<!-- Going into a subfolder --> <img src="images/gallery/photo.jpg" alt="Gallery photo"> <a href="pages/blog/post.html">Blog Post</a> <link rel="stylesheet" href="css/style.css">

Folder names first, then the filename: folder/subfolder/file.

Example 5: Parent Directory Navigation

<!-- From pages/blog/post.html --> <!-- Go up twice to reach root, then into images --> <img src="../../images/logo.png" alt="Logo"> <a href="../../about.html">About</a> <a href="../../index.html">Home</a>

../ = up one level. ../../ = up two levels.

Example 6: Root-Relative Paths

<!-- Root-relative starts from domain root --> <!-- Same from ANY page on the site --> <img src="/images/logo.png" alt="Logo"> <a href="/pages/about.html">About</a> <link rel="stylesheet" href="/css/style.css">
Tip: Root-relative paths (starting with /) are great for navigation menus and shared resources because they work from any page depth.

Example 7: Absolute (External) Paths

<!-- Full URL including protocol --> <img src="https://cdn.example.com/images/banner.jpg" alt="Banner"> <a href="https://www.google.com">Google</a> <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Roboto">

Use absolute paths for external resources. Always include the https:// protocol.

Example 8: Common Pitfalls

<!-- WRONG: backslash (Windows style) --> <img src="images\photo.jpg"> <!-- ✗ --> <!-- CORRECT: forward slash --> <img src="images/photo.jpg"> <!-- ✓ --> <!-- WRONG: missing extension --> <img src="images/photo"> <!-- ✗ --> <!-- WRONG: spaces in filenames --> <img src="images/my photo.jpg"> <!-- ✗ --> <!-- CORRECT: hyphens instead of spaces --> <img src="images/my-photo.jpg"> <!-- ✓ --> <!-- WRONG: wrong number of ../ --> <img src="../../../images/logo.png"> <!-- goes too far up -->
