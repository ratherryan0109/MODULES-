Module 44: HTML Form Attributes — Examples

Example 1: required, placeholder, autofocus

required
placeholder
autofocus
<input type="text" required placeholder="John Doe" autofocus>
<input type="email" required placeholder="john@example.com">

Example 2: readonly vs disabled

Key difference: readonly values are submitted with the form; disabled values are NOT submitted.
<input type="text" name="username" value="john_doe" readonly>
<input type="text" name="role" value="Admin" disabled>

Example 3: maxlength and minlength

<input type="text" minlength="4" maxlength="10">
<textarea maxlength="100"></textarea>

Example 4: min, max, step

5 
<input type="number" min="1" max="99" step="1">
<input type="range" min="0" max="10" step="1">

Example 5: pattern (RegEx Validation)

<input type="text" pattern="[0-9]{5}" title="Enter 5-digit ZIP code">
<input type="tel" pattern="[+][0-9]{1,3}[0-9]{6,14}">
The pattern attribute accepts a JavaScript regex. The title attribute provides the error message.

Example 6: formaction, formmethod, formtarget

<button type="submit" formaction="/preview" formmethod="GET">
<button type="submit" formnovalidate>
<button type="submit" formtarget="_blank">

Example 7: autocomplete Attributes

<input type="text" autocomplete="name">
<input type="email" autocomplete="email">
<input type="text" autocomplete="street-address">
Autocomplete values give browsers hints for auto-filling. See full list of tokens on MDN.

Example 8: form Attribute (Input Outside Form)

<form id="form8" action="/submit">...</form>
<!-- This input is OUTSIDE the <form> but submits with it: -->
<input type="text" name="outside" form="form8">
Summary: Form attributes control validation (required, pattern, minlength, maxlength), submission (formaction, formmethod, formtarget), behavior (autocomplete, novalidate), and state (readonly, disabled). Use them together for powerful, accessible forms.
