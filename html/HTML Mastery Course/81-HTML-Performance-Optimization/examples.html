HTML Performance Optimization - Live Examples

Open DevTools Network tab to see the performance impact of these techniques.

Note: These examples demonstrate HTML patterns. Actual performance gains depend on your server, network, and specific implementation.

Example 1: Resource Hints

Resource hints tell the browser to perform actions early, improving perceived performance.

<!-- Preconnect: Establish early connections to origins -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://cdn.example.com" crossorigin>

<!-- DNS-Prefetch: Resolve DNS early (broader support) -->
<link rel="dns-prefetch" href="https://analytics.example.com">

<!-- Preload: Load critical resources urgently -->
<link rel="preload" href="hero.webp" as="image">
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

<!-- Prefetch: Load resources for next page (idle time) -->
<link rel="prefetch" href="/next-page/" as="document">
When to use each: preload = critical current page resources, preconnect = third-party origins, dns-prefetch = fallback for older browsers, prefetch = likely next page resources.

Example 2: Critical CSS Inlining

Inline above-the-fold CSS to eliminate render-blocking requests.

❌ Bad: All CSS is render-blocking: 
<link rel="stylesheet" href="styles.css">
<link rel="stylesheet" href="components.css">
<link rel="stylesheet" href="utilities.css">
✅ Good: Critical CSS inlined, rest loaded async: 
<style>
  /* Critical above-the-fold styles */
  header { background: #2c3e50; color: white; padding: 1rem; }
  .hero { font-size: 2rem; text-align: center; padding: 4rem 1rem; }
  .hero h1 { margin-bottom: 1rem; }
</style>

<!-- Load non-critical CSS asynchronously -->
<link rel="preload" href="styles.css" as="style" 
      onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="styles.css"></noscript>
Header (critical CSS - renders immediately)

Hero Title

Above-the-fold content is visible immediately while full CSS loads.

Example 3: Image Optimization Techniques

❌ Bad: No dimensions, no lazy loading: 
<img src="photo.jpg" alt="Photo">

❌ Causes CLS (no dimensions), loads immediately (even offscreen), no optimization

✅ Good: Fully optimized: 
<img src="photo.webp" 
     srcset="photo-400.webp 400w, photo-800.webp 800w, photo-1200.webp 1200w"
     sizes="(max-width: 600px) 100vw, 800px"
     alt="Descriptive text"
     width="800" height="600"
     loading="lazy" 
     decoding="async">

✅ Prevents CLS, lazy loads, responsive, modern format

Notice: space is reserved even before image loads (no layout shift)

Example 4: Script Loading Comparison

<!-- Blocking: Parses, downloads, and executes before continuing -->
<script src="heavy.js"></script>

<!-- Defer: Downloads while parsing, executes after parsing (in order) -->
<script src="vendor.js" defer></script>
<script src="app.js" defer></script>

<!-- Async: Downloads while parsing, executes as soon as available (out of order) -->
<script src="analytics.js" async></script>
<script src="widget.js" async></script>

<!-- Module: ES6 module, deferred by default -->
<script type="module" src="main.mjs"></script>
Method Download Execution Order Use Case
No attribute Blocks HTML parsing Immediately when ready In order Avoid unless necessary
defer Parallel with parsing After HTML parsed In order Scripts that need DOM
async Parallel with parsing As soon as downloaded Any order Independent scripts (analytics)
type="module" Parallel with parsing After HTML parsed In order ES6 modules

Example 5: Font Optimization

❌ Bad: Synchronous font loading causes FOIT (Flash of Invisible Text): 
<link href="https://fonts.googleapis.com/css2?family=Roboto" rel="stylesheet">
✅ Good: Preload fonts + font-display: swap: 
<!-- Preconnect to Google Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- Preload the woff2 file -->
<link rel="preload" href="/fonts/inter-var.woff2" as="font" type="font/woff2" crossorigin>

<style>
  @font-face {
    font-family: 'Inter';
    src: url('/fonts/inter-var.woff2') format('woff2');
    font-display: swap; /* Show fallback text immediately, swap when font loads */
    font-weight: 400 700;
  }
</style>
font-display options: auto (browser default), block (short FOIT), swap (use fallback, swap when ready), fallback (short FOIT + swap), optional (short FOIT, no swap).

Example 6: Preventing Cumulative Layout Shift (CLS)

❌ Causes CLS: 
<img src="image.jpg" alt="Description">
<!-- Browser doesn't know dimensions until image loads -->
<!-- Content below shifts down when image loads -->

Text that shifts when image loads (no dimensions set)

✅ No CLS: 
<img src="image.jpg" alt="Description" width="800" height="450" style="max-width: 100%; height: auto;">

Text stays in place (browser reserves space from width/height)

Example 7: Responsive Images with srcset

Serve different image sizes based on screen width to save bandwidth.

<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w, 
             photo-800.jpg 800w, 
             photo-1200.jpg 1200w,
             photo-1600.jpg 1600w"
     sizes="(max-width: 600px) 100vw, 
            (max-width: 1024px) 50vw, 
            800px"
     alt="Landscape photo"
     width="800" height="600"
     loading="lazy">
How it works: Browser chooses the best image based on screen width, pixel density, and network conditions. 400w = 400 pixels wide. sizes tells the browser how much space the image will take.

Example 8: Performance Optimized HTML Skeleton

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Performance Optimized</title>
  <meta name="description" content="...">
  
  <!-- Preconnect to origins -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://cdn.example.com" crossorigin>
  
  <!-- Preload critical hero image & fonts -->
  <link rel="preload" href="hero.webp" as="image">
  <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
  
  <!-- Inlined critical CSS -->
  <style>
    /* Above-the-fold styles only */
    body { margin: 0; font-family: sans-serif; }
    header { background: #333; color: #fff; padding: 1rem; }
    .hero { max-width: 1200px; margin: 0 auto; padding: 4rem 1rem; }
  </style>
  
  <!-- Async non-critical CSS -->
  <link rel="preload" href="styles.css" as="style" 
        onload="this.onload=null;this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="styles.css"></noscript>
</head>
<body>
  <header>
    <img src="logo.svg" alt="Logo" width="100" height="40">
    <nav>...</nav>
  </header>
  
  <main>
    <section class="hero">
      <h1>Optimized Page</h1>
      <img src="hero.webp" alt="Hero" width="1200" height="600"
           srcset="hero-600.webp 600w, hero-1200.webp 1200w"
           sizes="100vw"
           loading="eager" decoding="async">
    </section>
    
    <section>
      <img src="gallery-1.webp" alt="Gallery" width="400" height="300"
           loading="lazy" decoding="async">
    </section>
  </main>
  
  <!-- Deferred scripts -->
  <script src="vendor.js" defer></script>
  <script src="app.js" defer></script>
</body>
</html>
