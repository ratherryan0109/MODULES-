Compare semantic vs non-semantic implementations side by side. Test with a screen reader to hear the difference.

1. Button vs div click handler

Non-semantic
Click me (div)

Not keyboard accessible, no role, no focus

Semantic

Keyboard accessible, focusable, proper role

<!-- Non-semantic --> <div onclick="alert('Clicked')">Click me</div> <!-- Semantic --> <button onclick="alert('Clicked')">Click me</button>

7. Details/Summary (Native Disclosure)

Native semantic disclosure widget - no JavaScript needed:

What is semantic HTML?

Semantic HTML uses elements that convey meaning about their content, such as <nav>, <article>, and <form>, instead of generic <div> elements. This improves accessibility, SEO, and code maintainability.

Why is it important?

Semantic HTML provides built-in accessibility features, better screen reader support, and improved keyboard navigation without requiring ARIA attributes or JavaScript.

<details> <summary>Question title</summary> <p>Answer content</p> </details> <!-- Built-in: keyboard toggle with Enter/Space -->
