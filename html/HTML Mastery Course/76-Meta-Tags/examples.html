Module 76: Meta Tags — Examples

Note: Most meta tags are not visible on the page. Use your browser's Developer Tools (F12) and inspect the <head> section to see them. This page itself contains meta tags in its head — check the page source!

Example 1: Essential Meta Tags for Every Page

These meta tags should appear on every HTML5 page.

<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Must be first -->
    <meta charset="UTF-8">

    <!-- Mobile responsiveness -->
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">

    <!-- SEO -->
    <meta name="description"
          content="Learn HTML meta tags with examples">

    <!-- Page title -->
    <title>Meta Tags Guide</title>
</head>
<body>
    <h1>Meta Tags Demo</h1>
    <p>Check the page source to see the meta tags.</p>
</body>
</html>
The meta tags above are in use on this very page! Open DevTools to inspect them.

Example 2: SEO Meta Tags

Meta tags that influence search engine ranking and display.

<!-- Page description (appears in search results) -->
<meta name="description"
      content="Master HTML meta tags: charset, viewport, SEO, robots, and social media. Complete guide with examples.">

<!-- Author credit -->
<meta name="author" content="HTML Mastery Course">

<!-- Search engine indexing -->
<meta name="robots" content="index, follow">

<!-- Google-specific -->
<meta name="googlebot" content="index, follow, max-snippet:-1, max-image-preview:large">

These tags are not visible but tell search engines how to index this page.

Example 3: Viewport Meta Tag Variations

Different viewport configurations for different use cases.

<!-- Standard responsive viewport -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

<!-- Fixed width (non-responsive) -->
<meta name="viewport"
      content="width=1024">

<!-- Disable zooming (use with caution) -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">

<!-- Allow zoom with limits -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0, minimum-scale=0.5, maximum-scale=3.0">
Viewport Parameters:
ParameterValuesDescription
widthdevice-width or pixelsWidth of the viewport
initial-scale0.1 - 10.0Initial zoom level
minimum-scale0.1 - 10.0Minimum zoom
maximum-scale0.1 - 10.0Maximum zoom
user-scalableyes / noAllow user zoom

Example 4: Robots Meta Tag Directives

Control how search engines interact with your page.

<!-- Allow full indexing -->
<meta name="robots" content="index, follow">

<!-- Prevent indexing but follow links -->
<meta name="robots" content="noindex, follow">

<!-- Prevent everything -->
<meta name="robots" content="noindex, nofollow">

<!-- Prevent snippet/cache -->
<meta name="robots" content="index, follow, noarchive, nosnippet">

<!-- Advanced Google directives -->
<meta name="googlebot" content="noimageindex, max-video-preview:-1">
Directive Reference:
DirectiveEffect
indexAllow page to be indexed
noindexPrevent indexing
followFollow links on page
nofollowDo not follow links
noarchiveDo not cache page copy
nosnippetDo not show text snippet
noimageindexDo not index images

Example 5: Refresh and Redirect Meta Tags

Use http-equiv to simulate HTTP headers.

<!-- Auto-refresh every 30 seconds -->
<meta http-equiv="refresh" content="30">

<!-- Redirect after 5 seconds -->
<meta http-equiv="refresh" content="5; url=https://example.com">

<!-- X-UA-Compatible for IE -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- Content-Type declaration (redundant with charset) -->
<meta http-equiv="Content-Type"
      content="text/html; charset=UTF-8">

<!-- CSP (Content Security Policy) -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self' https://analytics.example.com">
http-equiv Reference:
http-equivPurpose
refreshAuto-refresh or redirect
X-UA-CompatibleIE rendering mode
Content-TypeMIME type and charset
Content-Security-PolicySecurity policies
default-stylePreferred stylesheet

Example 6: Theme Color and Mobile Meta Tags

Customize the browser chrome on mobile devices.

<!-- Theme color (Chrome, Firefox, Edge on mobile) -->
<meta name="theme-color" content="#e44d26">

<!-- Apple-specific -->
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="Meta Guide">

<!-- Windows Tile -->
<meta name="msapplication-TileColor" content="#e44d26">
<meta name="msapplication-TileImage" content="/icon-144.png">

<!-- Application Name -->
<meta name="application-name" content="HTML Meta Tags Guide">
Result:

If viewing on Chrome for Android, the toolbar above may match the theme color. On iOS, the status bar style changes.

Meta TagPlatform
theme-colorChrome, Firefox, Edge
apple-mobile-web-app-capableSafari iOS
msapplication-TileColorWindows/Edge

Example 7: Social Media Meta Tags (Open Graph)

Control how your page appears when shared on social media platforms.

<!-- Open Graph (Facebook, LinkedIn, Pinterest) -->
<meta property="og:title" content="HTML Meta Tags Complete Guide">
<meta property="og:description"
      content="Learn everything about HTML meta tags: charset, viewport, SEO, and social media tags.">
<meta property="og:image" content="https://example.com/og-image.jpg">
<meta property="og:url" content="https://example.com/meta-tags-guide">
<meta property="og:type" content="article">
<meta property="og:site_name" content="HTML Mastery Course">
<meta property="og:locale" content="en_US">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="HTML Meta Tags Complete Guide">
<meta name="twitter:description"
      content="Learn everything about HTML meta tags.">
<meta name="twitter:image" content="https://example.com/twitter-image.jpg">

Example 8: Complete Production Head Section

A comprehensive head section with all important meta tags.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">

    <!-- SEO -->
    <title>HTML Meta Tags Guide | HTML Mastery</title>
    <meta name="description"
          content="Complete guide to HTML meta tags. Learn charset, viewport, SEO, robots, Open Graph, and Twitter Card meta tags.">
    <meta name="robots" content="index, follow">
    <meta name="author" content="HTML Mastery Course">

    <!-- Open Graph -->
    <meta property="og:title" content="HTML Meta Tags Guide">
    <meta property="og:description"
          content="Complete guide to HTML meta tags.">
    <meta property="og:image"
          content="https://example.com/og-image.jpg">
    <meta property="og:url"
          content="https://example.com/meta-tags">
    <meta property="og:type" content="article">

    <!-- Twitter -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="HTML Meta Tags Guide">
    <meta name="twitter:description"
          content="Complete guide to HTML meta tags.">
    <meta name="twitter:image"
          content="https://example.com/twitter-image.jpg">

    <!-- Mobile -->
    <meta name="theme-color" content="#e44d26">
    <meta name="apple-mobile-web-app-capable" content="yes">

    <!-- Canonical -->
    <link rel="canonical"
          href="https://example.com/meta-tags">

    <!-- Preconnect for performance -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
</head>
<body>
    <!-- Page content -->
</body>
</html>
Key Takeaway:

A production-ready head section includes charset, viewport, title, description, robots, Open Graph, Twitter Card, theme-color, and canonical URL. Order matters — put charset first.

Pro Tip: Use Google's Rich Results Test (search.google.com/test/rich-results) and Facebook Sharing Debugger (developers.facebook.com/tools/debug) to validate your meta tags.
