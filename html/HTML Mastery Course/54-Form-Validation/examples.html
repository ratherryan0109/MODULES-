Form Validation — Interactive Examples

1. Required Field

The required attribute prevents submission when the field is empty.

<input type="text" required>

2. Input Type Validation

HTML5 types like email, url, and tel provide built-in format validation.

3. Pattern Attribute

The pattern attribute uses regex to validate input formats.

<input type="text" pattern="[A-Za-z ]+" title="Letters and spaces only">
Note: The title attribute provides the error message when the pattern doesn't match.

4. Range and Length Constraints

min, max, minlength, maxlength, and step constrain numeric and text values.

5. Custom Validity with JavaScript

Use setCustomValidity() to override default error messages.

6. Password Confirmation

Custom validation logic ensures two passwords match.

7. CSS Validation Styling

CSS pseudo-classes :valid, :invalid, and :focus style inputs based on their validation state.

8. Real-Time Validation with Blur Events

Validate on blur (when the user leaves the field) to provide feedback without being intrusive.

9. ValidityState Inspection

Inspect the ValidityState object to see which validation rules are failing.

10. Complete Registration with All Validation

A comprehensive registration form combining all validation techniques.
