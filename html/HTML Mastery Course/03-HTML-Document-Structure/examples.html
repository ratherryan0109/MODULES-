Module 3: HTML Document Structure — Examples

Example 1: Complete HTML Document Anatomy

Every HTML document has this exact structure. Notice the DOCTYPE, html, head, and body.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A sample page for learning HTML structure">
    <title>Document Structure Example</title>
    <link rel="icon" href="favicon.ico">
</head>
<body>
    <h1>Hello, World!</h1>
    <p>Welcome to learning HTML document structure.</p>
</body>
</html>
Rendered Output:

Hello, World!

Welcome to learning HTML document structure.

Example 2: Meta Tags in the Head

Meta tags provide information about the page. They are not visible to users but crucial for SEO and functionality.

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Learn HTML document structure with examples">
    <meta name="keywords" content="HTML, structure, web development">
    <meta name="author" content="HTML Mastery Course">
    <meta property="og:title" content="HTML Document Structure">
    <meta property="og:description" content="Complete guide to HTML document anatomy">
    <title>Meta Tags Demo</title>
</head>
Note:

These meta tags are not visible on the page but affect how search engines and social platforms interpret this page.

Example 3: Semantic Page Layout

Using HTML5 semantic elements creates a meaningful document structure.

<body>
    <header>
        <h1>My Website</h1>
        <nav>
            <a href="#">Home</a>
            <a href="#">About</a>
        </nav>
    </header>

    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content...</p>
        </article>

        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>&copy; 2026 My Website</p>
    </footer>
</body>
Rendered Output:

My Website

Home | About

Article Title

Article content...

Related Links

  • Link 1

© 2026 My Website

Example 4: DOM Tree Visualization

The browser parses HTML into a tree structure called the Document Object Model.

Document
└── <html lang="en">
    ├── <head>
    │   ├── <meta charset="UTF-8">
    │   └── <title>DOM Demo</title>
    └── <body>
         ├── <h1>DOM Tree</h1>
         └── <p>Each element is a node.</p> 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>DOM Demo</title>
</head>
<body>
    <h1>DOM Tree</h1>
    <p>Each element is a node.</p>
</body>
</html>
Rendered Output:

DOM Tree

Each element is a node.

Example 5: Proper Nesting vs Improper Nesting

<!-- ✅ CORRECT: Inner tags close before outer -->
<div>
    <p>This is <strong>correct</strong> nesting.</p>
</div>

<!-- ❌ INCORRECT: Overlapping tags -->
<div>
    <p>This is <strong>incorrect</p> nesting.</strong>
</div>
Correct version renders properly:

This is correct nesting.

Example 6: The Importance of DOCTYPE

Without <!DOCTYPE html>, browsers switch to "quirks mode," causing inconsistent rendering.

<!-- Without DOCTYPE -->
<html>
<head>
    <title>Bad Practice</title>
</head>
<body>
    <p>This page has no DOCTYPE.</p>
</body>
</html>
