Clean Code Standards - Before & After Examples

Each example shows "bad" code alongside the clean, maintainable version.

Example 1: Consistent Indentation

❌ Bad: Inconsistent indentation 
<ul>
  <li>Item 1
  <li>Item 2
      <ul>
     <li>Sub-item A</li>
    <li>Sub-item B</li>
      </ul>
  </li>
  <li>Item 3</li>
</ul>
✅ Good: Consistent 2-space indentation 
<ul>
  <li>
    Item 1
  </li>
  <li>
    Item 2
    <ul>
      <li>Sub-item A</li>
      <li>Sub-item B</li>
    </ul>
  </li>
  <li>Item 3</li>
</ul>
Rule: Use 2-space indentation consistently. Each nesting level adds one indentation level. Close tags at the same level as their opening tag.

Example 2: BEM Naming Convention

❌ Bad: Unclear class names 
<div class="box">
  <div class="title">My Card</div>
  <div class="content">
    <p class="text">Description</p>
    <a class="link" href="#">Read</a>
  </div>
</div>

<div class="box">
  <div class="title">Another Card</div>
  <div class="content">
    <p class="text">More text</p>
    <a class="link special" href="#">Buy</a>
  </div>
</div>
✅ Good: BEM naming 
<article class="card">
  <header class="card__header">
    <h2 class="card__title">My Card</h2>
  </header>
  <div class="card__body">
    <p class="card__text">Description</p>
  </div>
  <footer class="card__footer">
    <a href="#" class="card__link">Read</a>
  </footer>
</article>

<article class="card">
  <header class="card__header">
    <h2 class="card__title">Another Card</h2>
  </header>
  <div class="card__body">
    <p class="card__text">More text</p>
  </div>
  <footer class="card__footer">
    <a href="#" class="card__link card__link--primary">Buy</a>
  </footer>
</article>
BEM structure: Block__Element--Modifier. Example: card__button--primary = block "card", element "button", modifier "primary".

Example 3: Separation of Concerns

❌ Bad: Mixed concerns 
<button style="background: blue; color: white; padding: 10px 20px; border: none; border-radius: 4px; font-size: 16px; cursor: pointer;" 
        onclick="submitForm(); return false;" 
        onmouseover="this.style.background='darkblue'" 
        onmouseout="this.style.background='blue'">
  Submit
</button>
❌ HTML + CSS + JS all mixed together. Hard to maintain, no reusability.
✅ Good: Separated concerns 
<button class="btn btn--primary" type="submit">
  Submit
</button>

<!-- CSS -->
<style>
  .btn {
    padding: 10px 20px;
    border: none;
    border-radius: 4px;
    font-size: 16px;
    cursor: pointer;
  }
  .btn--primary { background: #3498db; color: white; }
  .btn--primary:hover { background: #2980b9; }
</style>

<!-- JavaScript -->
<script>
  document.querySelector('.btn--primary')
    .addEventListener('click', submitForm);
</script>
✅ Clean separation: HTML (structure), CSS (style), JS (behavior).

❌ Bad: Div Soup 
<div class="header">
  <div class="logo">Site</div>
  <div class="nav">

  </div>
</div>

</footer>

Example 5: Organized Code with Comments

✅ Well-organized HTML structure: 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title - Site</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>

<!-- ========================================
     HEADER
     ======================================== -->
<header class="site-header">
  <a href="/" class="site-header__logo">
    <img src="logo.svg" alt="Site Name">
  </a>
  
  <!-- Primary Navigation -->
  <nav class="nav" aria-label="Main">
    <ul class="nav__list">
      <li class="nav__item"><a href="/" class="nav__link">Home</a></li>
      <li class="nav__item"><a href="/about" class="nav__link">About</a></li>
    </ul>
  </nav>
</header>
<!-- /HEADER -->

<!-- ========================================
     MAIN CONTENT
     ======================================== -->
<main class="main-content">
  <article class="post">
    <h1 class="post__title">Article Title</h1>
    <p class="post__meta">Published: <time datetime="2026-06-12">June 12, 2026</time></p>
    <div class="post__content">
      <p>Article content...</p>
    </div>
  </article>
</main>

<!-- ========================================
     FOOTER
     ======================================== -->
<footer class="site-footer">
  <p>&copy; 2026 Site Name. All rights reserved.</p>
</footer>
<!-- /FOOTER -->

</body>
</html>
Commenting guidelines: Use section comments (===) to separate major sections. Use /SECTION to close. Avoid inline comments for obvious code. Comment WHY, not WHAT.

Example 6: Using Data Attributes Properly

❌ Bad: Inline event handlers and unclear state 
<div id="myModal" class="modal" style="display:none;">
  <div class="modal-content">
    <span onclick="closeModal()">&times;</span>
    <p>Modal content</p>
  </div>
</div>
<button onclick="openModal()">Open</button>
✅ Good: Data attributes and event listeners 
<div id="modal" class="modal" 
     data-modal="confirmation" 
     data-visible="false"
     aria-hidden="true">
  <div class="modal__content">
    <button class="modal__close" 
            data-action="close-modal" 
            aria-label="Close modal">
      &times;
    </button>
    <p>Modal content</p>
  </div>
</div>
<button data-action="open-modal" 
        data-modal-id="modal">
  Open
</button>

<!-- JavaScript handles behavior -->
<script>
  const modal = document.getElementById('modal');
  document.querySelector('[data-action="open-modal"]')
    .addEventListener('click', () => {
      modal.dataset.visible = 'true';
      modal.setAttribute('aria-hidden', 'false');
    });
</script>

Example 7: DRY (Don't Repeat Yourself)

❌ Bad: Repeated code 
<div class="product-card">
  <img src="product-1.jpg" alt="Product 1">
  <h3>Product 1</h3>
  <p class="price">$19.99</p>
  <button>Add to Cart</button>
</div>
<div class="product-card">
  <img src="product-2.jpg" alt="Product 2">
  <h3>Product 2</h3>
  <p class="price">$29.99</p>
  <button>Add to Cart</button>
</div>
<div class="product-card">
  <img src="product-3.jpg" alt="Product 3">
  <h3>Product 3</h3>
  <p class="price">$39.99</p>
  <button>Add to Cart</button>
</div>
✅ Good: Use templating (server-side or JS) 
<!-- Using JavaScript template -->
<script type="text/template" id="product-template">
  <article class="product-card">
    <img src="{{image}}" alt="{{name}}">
    <h3 class="product-card__name">{{name}}</h3>
    <p class="product-card__price">\${{price}}</p>
    <button class="product-card__add-btn">Add to Cart</button>
  </article>
</script>

<div id="products-container"></div>

<script>
  const products = [
    { image: 'product-1.jpg', name: 'Product 1', price: '19.99' },
    { image: 'product-2.jpg', name: 'Product 2', price: '29.99' },
    { image: 'product-3.jpg', name: 'Product 3', price: '39.99' }
  ];
  
  const template = document.getElementById('product-template').innerHTML;
  const container = document.getElementById('products-container');
  
  products.forEach(product => {
    container.innerHTML += template
      .replace(/\{\{image\}\}/g, product.image)
      .replace(/\{\{name\}\}/g, product.name)
      .replace(/\{\{price\}\}/g, product.price);
  });
</script>

Example 8: EditorConfig Configuration

# .editorconfig - Place at project root
root = true

[*]
indent_style = space
indent_size = 2
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

[*.md]
trim_trailing_whitespace = false

[*.html]
indent_size = 2

[*.css]
indent_size = 2

[*.{json,yml,yaml}]
indent_size = 2
Why EditorConfig: Automatically maintains consistent coding styles across different editors and IDEs. Works with VS Code, Sublime, WebStorm, and most editors via plugins.
