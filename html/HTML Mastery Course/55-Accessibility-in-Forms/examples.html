2. Fieldset & Legend for Radio Groups

Radio buttons must be wrapped in <fieldset> with <legend> for screen reader context.

Preferred Contact Method

Without fieldset, the screen reader hears "Email radio button, Phone radio button, Mail radio button" with no context about what these options mean.

3. Help Text with aria-describedby

Use aria-describedby to associate help text and error messages with inputs.

<label for="password">Password</label>
<input type="password" id="password" aria-describedby="pw-hint pw-error">
<span id="pw-hint">8+ characters with a number</span>
<span id="pw-error" role="alert"></span>
Must be at least 8 characters with at least one number

4. Accessible Error Messages

Errors should be associated via aria-describedby, use role="alert", and include aria-invalid.

Enter your work email Please enter a valid email address

6. Autocomplete for Accessibility (WCAG 1.3.5)

The autocomplete attribute helps users with cognitive disabilities and enables browser auto-fill.

7. Live Regions for Dynamic Content

aria-live="polite" announces dynamic form updates without interrupting the current task.

0 of 200 characters used
Screen readers automatically announce the character count update because aria-live="polite" is set.

8. Keyboard Navigation Pattern

Custom controls must support keyboard interactions: Tab to focus, Enter/Space to activate, Arrow keys for selection.

Custom rating widget:

Rate your experience ☆ ☆ ☆ ☆ ☆

No rating selected

Use Tab to focus the stars, then Arrow keys to navigate, Space/Enter to select. Works keyboard-only.

9. Accessible Error Summary

After form submission with errors, move focus to the error summary and provide links to specific fields.

    Valid email is required
