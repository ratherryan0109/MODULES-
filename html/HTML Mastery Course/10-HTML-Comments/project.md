# Mini Project: Build a 'Documentation with Comments' Page

## Objective
Create a well-documented HTML page that demonstrates proper comment usage while teaching the user about HTML forms. Every major section should have section comments, complex parts should have explanatory comments, and there should be TODO and FIXME comments marking areas for improvement.

## Requirements

Build an HTML page about "HTML Forms" with:

1. **Documentation Comments at the Top**
   - File name, author, date, purpose, and version

2. **Section Markers**
   - Every section clearly marked with start/end comments

3. **Explanatory Comments**
   - Explain non-obvious code decisions
   - Why certain attributes are used
   - Accessibility considerations

4. **TODO Comments**
   - Mark at least 3 places where future improvements are needed

5. **FIXME Comments**
   - Mark at least 1 known limitation

6. **Content: HTML Forms Demo**
   - Include a contact form with text, email, textarea, select, and button
   - Show proper form structure and attributes

7. **Security Comment Section**
   - A visible (rendered) warning about not putting secrets in comments

## Step-by-Step Instructions

1. Create `documented-form.html`
2. Start with the documentation header comment
3. Build the contact form with comments throughout
4. Mark TODOs and FIXMEs
5. Add the security warning section
6. Validate the HTML

## Bonus Challenges
- Add a HACK comment showing a workaround
- Add a section that is temporarily disabled via comments
- Use comments to show the old vs new version of a feature
