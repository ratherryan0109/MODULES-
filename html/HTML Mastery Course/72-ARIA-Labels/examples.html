ARIA Labels - Interactive Examples

Each example demonstrates proper ARIA usage with live interactive demos. Open your browser's accessibility inspector to explore the accessibility tree.

1. Accessible Name Computation

These buttons all have the same visual appearance but different accessible name sources:

Close window
<!-- aria-label --> <button aria-label="Close dialog"> <span aria-hidden="true">×</span> </button> <!-- aria-labelledby --> <button aria-labelledby="close-label"> <span aria-hidden="true">×</span> </button> <span id="close-label" hidden>Close window</span> <!-- Fallback content --> <button> <span aria-hidden="true">×</span> <span>Dismiss</span> </button>

2. Tab Panel with ARIA

Accessible tab panel using role="tablist", role="tab", and role="tabpanel":

This premium widget features high-quality materials and ergonomic design. Perfect for everyday use.

  • Dimensions: 10 × 5 × 3 cm
  • Weight: 200g
  • Material: Aluminum alloy
  • Color: Space gray

★★★★☆ "Great product, highly recommended!"

★★★★★ "Exceeded my expectations."

<div role="tablist" aria-label="Product Information"> <button role="tab" aria-selected="true" aria-controls="panel-desc" id="tab-desc"> Description </button> <button role="tab" aria-selected="false" aria-controls="panel-specs" id="tab-specs"> Specifications </button> <button role="tab" aria-selected="false" aria-controls="panel-reviews" id="tab-reviews"> Reviews </button> </div> <div role="tabpanel" id="panel-desc" aria-labelledby="tab-desc"> <p>Content here...</p> </div>

3. Live Region - Chat Messages

New messages are announced via aria-live="polite":

System: Welcome to the chat!

<div aria-live="polite" aria-atomic="false" aria-relevant="additions" id="chat-log"> <p><strong>System:</strong> Welcome!</p> </div>

4. Custom Slider with ARIA

Accessible volume slider using role="slider":

50
<div role="slider" tabindex="0" aria-valuemin="0" aria-valuemax="100" aria-valuenow="50" aria-label="Volume" aria-orientation="horizontal"> <div class="thumb"></div> </div>

5. aria-describedby for Form Help

Input fields with additional descriptions:

We'll never share your email with third parties.

Format: name@example.com

  • At least 8 characters
  • One uppercase letter
  • One number or symbol
<label for="email">Email Address</label> <input type="email" id="email" aria-describedby="email-hint email-format"> <p id="email-hint">We'll never share...</p> <p id="email-format">Format: name@example.com</p>

6. aria-current for Navigation

Indicating the current page in navigation:

  1. Home
  2. /
  3. Products
  4. /
  5. Current Product
  • Home
  • Products
  • About
  • Contact
<a href="#" aria-current="page">Current Page</a> <!-- aria-current values: page, step, location, date, time, true -->

7. Modal Dialog with ARIA

Accessible modal using role="dialog" and aria-modal:

Confirm Deletion

Are you sure you want to delete this document? This action cannot be undone.

<div role="dialog" aria-modal="true" aria-labelledby="modal-title" aria-describedby="modal-desc"> <h2 id="modal-title">Confirm Deletion</h2> <p id="modal-desc">Are you sure?</p> </div>

8. aria-hidden and Icon Accessibility

Proper pattern for decorative icons:

<button aria-label="Add to favorites"> <svg aria-hidden="true" focusable="false"> <path d="..." /> </svg> </button>

9. Alert and Status Regions

Form validation with live region announcements:

<div role="alert" id="error"> <!-- Error messages announced immediately --> </div> <div role="status" id="status"> <!-- Status updates announced when idle --> </div>

10. Progress Bar with ARIA

Progress bar using role="progressbar":

0% complete
<div role="progressbar" aria-valuenow="0" aria-valuemin="0" aria-valuemax="100" aria-label="Upload progress"> <div style="width: 0%;"></div> </div>
