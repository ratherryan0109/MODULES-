# Mini Project: Build a Mobile-First Landing Page

## Objective
Create a mobile-first product landing page that looks great on all device sizes using HTML, viewport meta tag, and responsive CSS.

## Requirements

Build a responsive landing page with:

1. **Header** with sticky navigation
   - Logo and hamburger menu on mobile
   - Full horizontal nav on desktop

2. **Hero Section**
   - Full-width background image or gradient
   - Headline, subtext, and CTA button
   - Must fill viewport height on mobile

3. **Features Section** (3-column on desktop, stacked on mobile)
   - Icon/emoji, title, and description per feature
   - Smooth reveal on scroll

4. **Testimonials Section**
   - Card-based layout
   - 1 column on mobile, 2 on tablet, 3 on desktop

5. **Pricing Section**
   - 3 pricing tiers
   - Highlight "Popular" plan

6. **Contact Form**
   - Full-width on mobile
   - Centered within max-width on desktop

7. **Footer**
   - Link columns (stack on mobile)

## Responsive Breakpoints
- Mobile: < 768px (default)
- Tablet: 768px - 1024px
- Desktop: > 1024px

## Bonus Challenges
- Add CSS custom properties for theming
- Implement dark mode with `prefers-color-scheme`
- Add smooth scroll behavior
- Use `srcset` for responsive hero images

## Starter Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>ProductName - Landing Page</title>
</head>
<body>
    <!-- Build your mobile-first landing page -->
</body>
</html>
```

## Submission
Test on real mobile device or Chrome DevTools device emulation. Pass Lighthouse mobile audit.
