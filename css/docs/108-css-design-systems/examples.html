CSS Design Systems — Component Examples

1. Button Component (.c-btn)

Design system buttons using CSS custom properties for all values.

Link Button
.c-btn {\n background: var(--color-primary);\n color: white;\n padding: var(--spacing-sm) var(--spacing-md);\n border-radius: var(--radius-md);\n font-family: var(--font-sans);\n cursor: pointer;\n}

2. Card Component (.c-card)

Cards with BEM elements and consistent spacing tokens.

Card One
This card uses design system tokens for every visual property — spacing, color, border-radius, and typography.
Card Two
Because all values come from tokens, theming the entire system is as simple as changing the token values.
Card Three
Tokens ensure consistency across all components — the same spacing, colors, and typography everywhere.

3. Badge Component (.c-badge)

Small status indicators using semantic color tokens.

Success Warning Info Error
.c-badge--success { background: var(--color-success); }\n.c-badge--warning { background: var(--color-warning); }\n.c-badge--error { background: var(--color-error); }

4. Form Input (.c-input)

Consistent form elements using design tokens.

.c-input__field {\n border: 2px solid var(--color-border);\n border-radius: var(--radius-md);\n padding: var(--spacing-sm) var(--spacing-md);\n font-family: var(--font-sans);\n}\n.c-input__field:focus {\n border-color: var(--color-primary);\n}
