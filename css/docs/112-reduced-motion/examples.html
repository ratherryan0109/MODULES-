🎬 Reduced Motion & Accessible Animations

Learn how to design animations that respect user preferences

1. Decorative vs Functional Animation

❌ Decorative (Remove)

Bounce
Shake
🤹 Floating

Pure decoration — should be removed when motion is reduced.

✅ Functional (Keep)

Loading indicators and progress bars serve a purpose — keep them.

2. Notification Animation: Slide vs Fade

❌ Slide In (No Respect)

🔔 New message received!

✅ Fade In (Respects Preference)

🔔 New message received!

With reduced motion, this fades in instead of sliding.

3. Parallax: Fixed vs Scroll

Parallax Background

With reduced motion, background-attachment changes to scroll (static).

4. Hover Effects: Scale vs Static

Hover over me
Me too!

With reduced motion, hover scaling is replaced by a subtle shadow.

5. Skeleton Loading

The shimmer effect stops with reduced motion, leaving static placeholder blocks.

6. Full Reduction Demo

Enable "Reduce motion" in your OS settings or use the manual toggle button (top right). All animations on this page will respond accordingly.

System settings: Windows → Ease of Access → Display → Reduce motion. macOS → Accessibility → Display → Reduce motion.
