CSS Best Practices

Example 1: CSS Custom Properties for Consistency

Store design tokens as custom properties on :root for a single source of truth.

Token-based Card

All colors, spacing, and shadows come from var() references. Change one value in :root and it updates everywhere.

:root {
  --primary: #06b6d4;
  --radius: 8px;
  --space-lg: 2rem;
}
.card {
  padding: var(--space-lg);
  border-radius: var(--radius);
}

Example 2: Specificity Management

Keep selectors flat. Prefer classes over IDs. Avoid !important.

Bad

#sidebar ul li a { color: blue; }

High specificity — hard to override.

Good

.nav-link { color: blue; }

Low specificity — easy to extend.

/* Avoid */
#sidebar ul li a { color: blue; }

/* Prefer */
.nav-link { color: blue; }

Example 3: Defensive CSS — Overflow Handling

Always handle long text and large images gracefully.

Very long product name that should not break the layout

Supercalifragilisticexpialidocious

.ellipsis {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
img {
  max-width: 100%;
  height: auto;
  display: block;
}

Example 4: Performant Animations

Animate only transform and opacity to avoid triggering layout recalculations.

Hover

Uses transform: scale() and opacity only — no layout thrashing.

.box {
  transition: transform 0.3s, opacity 0.3s;
  /* will-change: transform;  for heavy animations */
}
.box:hover {
  transform: scale(1.15);
  opacity: 0.8;
}

Example 5: Accessible Focus Styles

Never remove outline without providing a visible replacement.

Bad: outline: none

Good: focus-visible

/* Never do this */
button:focus { outline: none; }

/* Do this instead */
button:focus-visible {
  outline: 3px solid #06b6d4;
  outline-offset: 2px;
}

Example 6: BEM Naming Convention

Block__Element--Modifier creates predictable, maintainable class names.

Default Card

Standard card with default styling.

Featured Card

Uses the --featured modifier.

.card { }             /* Block */
.card__title { }      /* Element */
.card__body { }       /* Element */
.card--featured { }   /* Modifier */

Example 7: CSS Reset / Normalize

A minimal reset ensures consistent box-sizing and removes default margins across browsers.

*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}
body {
  font-family: system-ui, sans-serif;
  line-height: 1.6;
}

Example 8: Responsive Units Over Pixels

Use rem, em, %, and vw/vh instead of px for better scalability.

Base text (1rem)

Small text (0.875rem)

Heading (1.25rem)

p { font-size: 1rem; }       /* 16px base */
small { font-size: 0.875rem; } /* 14px */
h3 { font-size: 1.25rem; }    /* 20px */
