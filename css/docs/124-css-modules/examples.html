CSS Modules

Example 1: What CSS Modules Do

CSS Modules automatically scope class names to components by adding a unique hash. This prevents naming conflicts.

/* Card.module.css (your source) */
.root { padding: 1.5rem; }
.title { font-size: 1.25rem; }

/* Compiled output (automatic) */
.Card_root_a1b2 { padding: 1.5rem; }
.Card_title_c3d4 { font-size: 1.25rem; }

/* React usage */
import styles from './Card.module.css';
<div className={styles.root}>
  <h2 className={styles.title}>Title</h2>
</div>

Scoped Card Component

This card's styles are locally scoped. The class names include a unique hash generated from the file path and contents.

/* Card.module.css */
.root { padding: 1.5rem; border-radius: 8px; }
.title { font-size: 1.25rem; font-weight: 600; }
.body { color: #64748b; }

/* React */
import styles from './Card.module.css';
export function Card() {
  return (
    <div className={styles.root}>
      <h3 className={styles.title}>Title</h3>
      <p className={styles.body}>Content</p>
    </div>
  );
}

Example 3: Composing Classes with composes

CSS Modules support a composes keyword to inherit styles from another class within the same module.

Inactive
/* Badge.module.css */
.base {
  display: inline-flex;
  align-items: center;
  padding: 0.25rem 0.75rem;
  border-radius: 9999px;
  font-size: 0.75rem;
}
.active {
  composes: base;
  background: #dcfce7;
  color: #166534;
}
.pending {
  composes: base;
  background: #fef9c3;
  color: #854d0e;
}

Example 4: Global Styles in CSS Modules

Use :global() to opt out of scoping for specific selectors that need to be global.

/* Typography.module.css */
.heading {
  font-size: 1.5rem;
  font-weight: 700;
}

/* Global reset — not scoped */
:global(body) {
  margin: 0;
  font-family: system-ui, sans-serif;
}

/* Global utility — not scoped */
:global(.sr-only) {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
}

Example 7: CSS Modules in Build Tools

CSS Modules work with Webpack (css-loader), Vite, Parcel, and Next.js. Enable with modules: true.

// webpack.config.js
module.exports = {
  module: {
    rules: [{
      test: /\.module\.css$/,
      use: [
        'style-loader',
        {
          loader: 'css-loader',
          options: { modules: true }
        }
      ]
    }]
  }
};

// vite.config.js (auto-enabled for *.module.css)
// Next.js (auto-enabled for *.module.css)

Example 8: Dev vs Production Class Names

In development, class names include the file path for debugging. In production, they're short hashes.

/* Development (readable) */
.Card_root_a1b2 { }
.Card_title_c3d4 { }

/* Production (short) */
._1a2b3c4d { }
._5e6f7g8h { }
