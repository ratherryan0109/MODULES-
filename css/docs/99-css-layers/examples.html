CSS Layers (@layer)

1. Layer Priority Visualization

Layers from lowest to highest priority. Later layers override earlier ones regardless of specificity.

1. Reset
(lowest)
2. Base
3. Components
4. Utilities
5. Overrides
(highest)
@layer reset, base, components, utilities, overrides;

/* Styles in later layers win over earlier layers,
   even if the earlier selector has higher specificity. */

2. Layer Declaration & Structure

Define layer order once, then assign styles to each layer. The order of @layer statements doesn't matter — only the initial declaration order.

@layer reset — Box-sizing, margin/padding reset
@layer base — Typography, colors, body defaults
@layer components — Buttons, cards, forms
@layer utilities — Spacing helpers, text alignment
@layer reset, base, components, utilities;

@layer reset {
  *, *::before, *::after { margin: 0; padding: 0; box-sizing: border-box; }
}

@layer base {
  body { font-family: system-ui; color: #0f172a; background: #f8fafc; }
  h1, h2, h3 { font-weight: 700; }
}

@layer components {
  .btn { padding: 8px 20px; border-radius: 6px; font-weight: 600; }
  .card { background: white; border: 1px solid #e2e8f0; border-radius: 8px; }
}

@layer utilities {
  .mt-4 { margin-top: 16px; }
  .text-center { text-align: center; }
}

3. Layer vs Specificity

Layers win over specificity. Notice how a low-specificity selector in a later layer overrides a high-specificity selector in an earlier layer.

Without Layers

/* High specificity */
#sidebar .widget a { color: red; }

/* Lower specificity — WON'T override */
.widget-link { color: blue; }

/* Result: red (high specificity wins) */

With Layers

@layer base, components;

@layer base {
  #sidebar .widget a { color: red; }
  /* specificity: 0,1,0,2 */
}

@layer components {
  .widget-link { color: blue; }
  /* specificity: 0,0,1,0 BUT later layer! */
}

/* Result: blue (components layer wins) */

4. Unlayered Styles vs Layers

Styles outside any @layer have lower priority than layered styles. This is the opposite of what many expect!

Unlayered styles have LOWER priority than layered styles. To override a layer, place styles in a later layer — not outside any layer. 

/* UNLAYERED — lowest priority */
body { color: red; }

@layer base {
  body { color: #0f172a; } /* wins over unlayered */
}

/* To override layers, add a later layer: */
@layer overrides {
  body { color: green; } /* wins over base */
}

5. !important Reverses Layer Order

When !important is used, earlier layers win over later layers — the reverse of normal behavior.

Normal: Later layer wins → Overrides > Base

With !important: Earlier layer wins → Base > Overrides

@layer base {
  .text { color: red !important; } /* this wins */
}

@layer overrides {
  .text { color: blue !important; }
}

/* With !important, earlier layers beat later layers */

6. Layered @import

Third-party CSS can be imported directly into a layer, preventing it from overriding your styles.

/* Third-party framework goes into 'framework' layer */
@import url('bootstrap.css') layer(framework);

/* Your components layer can override framework styles */
@layer components {
  .btn { background: #3b82f6; /* overrides bootstrap */ }
}

/* Multiple libraries in the same layer: */
@import url('reset.css') layer(base);
@import url('typography.css') layer(base);

7. Anonymous & Nested Layers

Layers can be anonymous or nested for fine-grained control over cascade ordering.

/* Anonymous layer — no name */
@layer {
  * { box-sizing: border-box; }
}

/* Nested layers */
@layer framework {
  @layer base {
    body { font-family: system-ui; }
  }
  @layer components {
    .btn { padding: 8px 16px; }
  }
  /* Nested layers override each other by declaration order */
}
