BEM Methodology

Example 1: Block — Card Component

The Block is the top-level component. .card is the block.

This is the .card block. It is a standalone, reusable component.

<div class="card">
  <p>This is the .card block.</p>
</div>

Example 2: Elements Inside a Block

Elements are parts of the block, named with double underscore: .card__title, .card__body.

Card Title

This is the card body element. Elements are always part of their block.

<article class="card">
  <h3 class="card__title">Card Title</h3>
  <p class="card__body">Card body content.</p>
  <div class="card__actions">
    <button class="btn">Read More</div>
  </div>
</article>

Example 3: Modifiers for Variations

Modifiers change the appearance or state of a block/element, named with double dash: .card--featured.

Default Card

Standard card styling.

Featured Card

Uses card--featured and card__title--large modifiers.

<div class="card card--featured">
  <h3 class="card__title card__title--large">
    Featured Card
  </h3>
</div>

Example 5: Media Object with BEM

A classic media object with figure and body as elements, with alignment modifiers.

Default Media

Image on the left, content on the right.

Centered + Reverse

Two modifiers combined: media--center and media--reverse.

.media { display: flex; gap: 1rem; }
.media__figure { width: 80px; height: 80px; }
.media__body { flex: 1; }
.media--center { align-items: center; }
.media--reverse { flex-direction: row-reverse; }

Example 6: BEM Naming Rules

Follow these rules to keep BEM clean and predictable.

Block: .block-name (lowercase, hyphens)

Element: .block__element-name (double underscore)

Modifier: .block--modifier or .block__element--modifier (double dash)

Wrong: .block__element__subelement (elements within elements — don't do this)

Wrong: .Block_Name (camelCase or underscores in blocks)

Wrong: .block__element--modifier--another (multiple dashes)

/* Correct */
.card { }
.card__title { }
.card--featured { }
.card__title--large { }

/* Incorrect */
.card__header__title { }  /* no element-of-element */
.Card_Title { }           /* no camelCase */

Example 7: BEM File Organization

Keep each block in its own file for maximum maintainability.

components/
├── card.css        /* .card, .card__title, .card--featured */
├── button.css      /* .btn, .btn--primary, .btn--small */
├── media.css       /* .media, .media__figure, .media--center */
└── nav.css         /* .nav, .nav__item, .nav--dark */
