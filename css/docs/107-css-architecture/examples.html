3. OOCSS: Separate Structure from Skin

The same .o-grid object works with different visual themes.

Dark surface
Alt surface
Lighter surface
/* Structure */\n.o-grid { display: grid; gap: 16px; }\n\n/* Skin */\n.theme-dark { background: #1e1e2e; color: #cdd6f4; }\n.theme-light { background: #fff; color: #333; }

4. SMACSS Categories

Prefix conventions help identify each rule's purpose at a glance.

/* Layout */\n.l-header { }\n.l-sidebar { }\n.l-main { }\n\n/* State */\n.is-active { }\n.is-hidden { }\n.is-disabled { }\n\n/* Module */\n.modal { }\n.nav { }\n\n/* Theme */\n.theme-dark { }

5. Design Tokens in Action

All components use CSS custom properties. Changing the tokens propagates everywhere.

--color-success --color-warning
:root {\n --color-primary: #667eea;\n --color-success: #a6e3a1;\n --color-warning: #f9e2af;\n /* One change updates everything */\n}

6. Specificity Comparison

❌ High specificity
#sidebar .widget ul li a {\n color: red;\n}\n/* specificity: 0,1,0,4 */\n/* Hard to override */
✅ Low specificity (flat)
.widget-link {\n color: red;\n}\n/* specificity: 0,0,1,0 */\n/* Easy to override */
