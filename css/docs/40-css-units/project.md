# Mini Project: Responsive Design System Prototype

## Goal
Build a responsive design system prototype that demonstrates proper use of various CSS units — typography scale, spacing system, layout grid, and full-screen components.

## Requirements
- Typography system using `rem` and `clamp()` for h1–h6, body, small, and captions
- Spacing system using `rem` for margin/padding (4px base → .25rem, .5rem, 1rem, 2rem, 4rem)
- Responsive grid layout using `fr`, `%`, and `minmax()`
- Full-screen hero using viewport units
- Container with `min()` for max-width constraint
- Cards using `ch` for readable line lengths
- `calc()` used for at least one layout adjustment

## Stretch Goals
- Use `dvh` (dynamic viewport height) for better mobile experience
- Create a CSS reset that uses relative units
- Add CSS custom properties for the unit scale

## Evaluation Criteria
- All major CSS unit types are demonstrated
- Typography is fluid and accessible
- Layout adapts without media queries where possible
- Code is clean and well-organized
- No hardcoded magic numbers without unit justification
