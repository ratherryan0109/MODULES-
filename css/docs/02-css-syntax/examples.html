CSS Syntax: Examples & Demonstrations

1. Basic CSS Rule Structure

Every CSS rule follows the pattern: selector { property: value; }

/* Selector → Declaration Block */
h1 {
  color: navy; /* property: value; */
  font-size: 32px; /* property: value; */
  text-align: center;/* property: value; */
}

This h1-like text is styled using that exact rule.

2. Correct vs Incorrect Syntax

Syntax Valid? Notes
p { color: red; } ✅ Valid Standard correct syntax
p { color: red; font-size: 16px; } ✅ Valid Multiple declarations with semicolons
p { color: red font-size: 16px; } ❌ Invalid Missing semicolon — font-size ignored
p { color = red; } ❌ Invalid Using = instead of :
P { COLOR: RED; } ✅ Valid Case-insensitive for properties/values
h1, h2, h3 { color: blue; } ✅ Valid Grouped selectors
margin: 10px 20px 10px 20px; ✅ Valid Shorthand margin (top right bottom left)
margin: 10px, 20px; ❌ Invalid Commas don't belong in margin shorthand

3. Shorthand vs Longhand Comparison

Longhand (Verbose)

.box {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;
  border-width: 2px;
  border-style: solid;
  border-color: #333;
  margin-top: 5px;
  margin-right: 10px;
  margin-bottom: 5px;
  margin-left: 10px;
}

Shorthand (Concise)

.box {
  padding: 10px 20px;
  border: 2px solid #333;
  margin: 5px 10px;
}
💡 Tip: Shorthand is concise but be careful — it resets omitted values to defaults. For example, background: #fff; resets background-image to none.

4. Vendor Prefixes

Experimental CSS features may need vendor prefixes for cross-browser support:

.box {
  -webkit-transform: rotate(45deg); /* Chrome, Safari */
  -moz-transform: rotate(45deg); /* Firefox */
  -ms-transform: rotate(45deg); /* IE/Edge */
  -o-transform: rotate(45deg); /* Older Opera */
  transform: rotate(45deg); /* Standard */
}
Rotated!

Always put the unprefixed standard property last so it takes precedence when supported.

5. Whitespace Flexibility

CSS is flexible with whitespace — all of these are equivalent:

Compact:
p{color:red;font-size:14px;}
Readable (recommended):
p {
  color: red;
  font-size: 14px;
}

6. When to Use Quotes

Situation Example Quotes Required?
Font family with spaces font-family: "Times New Roman", serif; ✅ Yes for "Times New Roman"
Content property .foo::before { content: "→ "; } ✅ Yes
URL in url() background: url("bg.jpg"); ⚠️ Optional (recommended)
Attribute selectors [data-type="primary"] ✅ Yes for the attribute value
Fonts without spaces font-family: Arial; ❌ No

7. The !important Annotation

Adding !important gives a declaration the highest priority:

#special { color: green; }
p { color: blue !important; }

<p id="special">What color am I?</p>

I'm blue because of !important, even though the ID selector would normally win.

⚠️ Warning: Use !important sparingly! It breaks the cascade and makes maintenance difficult.

8. Grouping Selectors

Use commas to apply the same rule to multiple selectors:

h1, h2, h3, .section-title {
  font-family: Georgia, serif;
  color: #2c3e50;
  margin-bottom: 0.5em;
}
