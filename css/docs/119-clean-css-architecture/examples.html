Clean CSS Architecture

Example 1: CSS Cascade Layers (ITCSS)

Six layers from low to high priority prevent specificity wars. Each layer has a clear responsibility.

@layer settings  { /* Variables, tokens        */ }
@layer generic   { /* Reset / normalize        */ }
@layer elements  { /* HTML element defaults    */ }
@layer objects   { /* Layout patterns          */ }
@layer components{ /* UI components (cards, btns) */ }
@layer utilities { /* Override helpers         */ }

Example 2: Settings Layer — Design Tokens

All colors, spacing, and typography values live here. No hard-coded values elsewhere.

@layer settings {
  :root {
    --primary: #06b6d4;
    --radius: 8px;
    --space: 1rem;
    --font: system-ui, sans-serif;
  }
}

Example 3: Generic Layer — Reset

Normalize box-sizing and remove default margins. This is the foundation.

@layer generic {
  *, *::before, *::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
  }
}

Example 5: Components Layer — UI Modules

Components are prefixed with c- and follow BEM for internal structure.

Featured Component

This card and button are in the @layer components layer.

Standard Component

Consistent structure with no surprise overrides.

@layer components {
  .c-card { background: #fff; border: 1px solid var(--border); }
  .c-card__title { font-size: 1.25rem; }
  .c-card--featured { border-color: var(--primary); }
  .c-btn { display: inline-flex; padding: 0.5rem 1rem; }
  .c-btn--primary { background: var(--primary); }
}

Example 6: Utilities Layer — Single-Purpose Overrides

Utilities are prefixed with u- and do one thing. They sit at the highest layer to override anything below.

This text is centered and muted using utility classes.

@layer utilities {
  .u-mt { margin-top: 1rem; }
  .u-text-center { text-align: center; }
  .u-text-muted { color: #64748b; }
}

Example 7: Why Layers Beat Specificity Hacks

Without layers, a utility class might not override a component style if the component uses a higher-specificity selector. Layers guarantee ordering regardless of specificity.

No Layers

nav ul li a { color: blue; !important }

Needs !important or higher specificity to override.

With Layers

@layer components { ... } @layer utilities { }

Utilities always win regardless of specificity.

@layer utilities {
  .u-text-center { text-align: center; }
}
/* Always wins against component-level styles */
