CSS Container Queries

1. Responsive Card with Container Query

Drag the bottom-right corner of the dashed container to resize it. The card layout switches from stacked to side-by-side when the container is wide enough.

Container Query Card

This card adapts to its container width, not the viewport! Resize the dashed container to see the layout change.

CSS Container Queries 
.card-wrap {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 360px) {
  .cq-card {
    grid-template-columns: 140px 1fr;
  }
}

@container card (min-width: 520px) {
  .cq-card {
    grid-template-columns: 180px 1fr;
  }
  .cq-card .content h3 { font-size: 1.3rem; }
}

3. Container Query Units (cqi)

Using cqi (container inline-size unit) for responsive typography that scales with the container. Resize to see the text grow and shrink.

Responsive Typography

This text uses clamp() with cqi units: font-size scales smoothly from 1rem to 2rem based on container width.

.card .title {
  font-size: clamp(1rem, 6cqi, 2rem);
  /* 6cqi = 6% of the container's inline-size */
}

.card .body {
  font-size: clamp(0.8rem, 3cqi, 1rem);
}

4. Sidebar + Main with Container Queries

Both the sidebar and main content areas are container query contexts. Each adapts independently to its own width.

Sidebar

Narrow container — stays stacked.

Main Content

Wider container — switches to side-by-side layout automatically.

Container Query 
.sidebar { container-type: inline-size; }
.main { container-type: inline-size; }
/* Each container independently responds to its width */

5. Container Query Style & Layout Toggle

A card that switches from vertical (centered) to horizontal (left-aligned) layout based on the container width — no media queries needed.

📦

Product Card

Resize container to see layout flip.

@container (min-width: 250px) {
  .cq-style-card {
    flex-direction: row; /* switches from column */
    text-align: left;
  }
}
