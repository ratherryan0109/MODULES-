Responsive Accessibility

Example 1: Fluid Typography with clamp()

Text scales between 1rem and 1.5rem based on viewport width, keeping readability at all sizes.

This heading text scales fluidly between 16px and 24px depending on your viewport width. Resize the browser to see it change.

Normal body text at a fixed size for comparison.

h2 {
  font-size: clamp(1rem, 3vw, 1.5rem);
}

Example 2: Responsive Card Grid with Touch Targets

Cards reflow into fewer columns on narrow screens. Each card and link has minimum 44×44px touch targets.

Dashboard

Monitor your stats at a glance.

Settings

Adjust your preferences.

Profile

View and edit your profile.

Reports

Download usage reports.

.card-grid {
  display: grid;
  grid-template-columns: repeat(
    auto-fill,
    minmax(min(200px, 100%), 1fr)
  );
  gap: 1rem;
}

Example 3: Accessible Responsive Navigation

On wide screens links are in a row. On narrow screens they wrap and each becomes full-width for easy tapping.

Home Products Services About Contact 
nav a {
  padding: 0.5rem 1rem;
  min-height: 44px;
  display: flex;
  align-items: center;
}
@media (max-width: 600px) {
  nav a { flex: 1; justify-content: center; }
}

Example 4: Mobile-first Form with Adequate Touch Targets

All form controls have min-height: 44px and large tap areas. Labels are always visible above inputs.

input, button {
  min-height: 44px;
  padding: 0.75rem;
  font-size: 1rem;
}
button {
  min-width: 44px;
}

Example 5: Horizontally Scrollable Table

Wide tables scroll horizontally on small screens instead of breaking layout. Table headers stay readable.

th, td {

}

Example 6: Zoom-safe Layout with Relative Units

Using rem/em units ensures the layout scales when the user zooms the page.

Zoom in (Ctrl/Cmd +)

This container uses rem-based padding and font sizes. Everything scales proportionally when you zoom.

.container {
  padding: 2rem;
  max-width: 40rem;
}
p { font-size: 1rem; }

Example 7: Screen Reader Only Text for Responsive Context

Extra text for screen readers can clarify context that changes visually at different breakpoints.

Status: ● Online - Connected

Screen readers hear "Online - Connected" while visual users see the green dot and "Connected".

.sr-only {
  position: absolute;
  width: 1px; height: 1px;
  overflow: hidden;
  clip: rect(0,0,0,0);
  white-space: nowrap;
}
