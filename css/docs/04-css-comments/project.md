# Mini Project: Commented Portfolio Stylesheet

## Project Overview
Take a CSS file and transform it into a well-documented, self-explanatory stylesheet through strategic commenting. The goal is to make the CSS understandable to another developer without them needing to see the HTML.

## Requirements
1. Start with a raw CSS file (at least 50 lines) with NO comments
2. Add a file-header comment block (description, author, date, version)
3. Add section headers for logical groupings (Typography, Layout, Components, etc.)
4. Add inline comments for non-obvious properties
5. Add TODO comments for at least 3 improvements
6. Add a commented-out fallback for an older browser
7. Add FIXME for at least 1 known issue

## Submission
Submit both the original (uncommented) and commented versions of the CSS file.

## Rubric
- File header block (10%)
- Logical section organization (20%)
- Inline explanations (non-obvious) (25%)
- TODO/FIXME/HACK markers (15%)
- Commented fallbacks (15%)
- Consistent formatting (15%)
