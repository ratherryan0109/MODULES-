CSS Introduction: Examples

1. Inline CSS Example

This paragraph is styled with inline CSS — notice the style attribute directly on the HTML tag.

Inline styles have the highest specificity (unless !important is used).

<p style="color: #e53e3e; font-weight: bold; font-size: 18px;"> This paragraph is styled with inline CSS... </p>

2. Internal CSS Example

The styles on this page are applied via an internal stylesheet inside the <style> tag in the document head.

All the styling you see (fonts, colors, borders, layouts) is from the internal CSS block at the top of this HTML file.

<head> <style> body { font-family: 'Segoe UI', sans-serif; background-color: #f0f4f8; color: #2d3748; } h1 { color: #1a365d; } </style> </head>

3. External CSS Reference

In a real project, you would use a <link> tag in the HTML <head> to connect an external stylesheet:

<head> <link rel="stylesheet" href="styles.css"> </head>

The external approach is best for production because:

✅ External

  • Reusable across pages
  • Browser-cacheable
  • Clean separation
  • Easy maintenance

⚠️ Internal

  • Single page only
  • No caching benefit
  • Mixes with HTML
  • Harder to maintain

❌ Inline

  • Not reusable
  • Hard to maintain
  • Highest specificity
  • Use sparingly

4. The Cascade in Action

CSS rules are applied based on cascade rules. Here, three rules target the same paragraph:

This paragraph should be green because the last rule wins in the stylesheet.

But this one is blue because the class selector has higher specificity.

The code below shows three conflicting rules. The cascade resolves them:

.cascade-demo p { color: #e53e3e; } /* Red - overridden */ .cascade-demo .special { color: #3182ce; } /* Blue - wins for .special */ .cascade-demo p { color: #38a169; } /* Green - wins for plain p (last) */

5. Specificity Demonstration

Specificity determines which rule applies when selectors conflict:

What color am I? I have an ID (#specific-id), a class (.specific-class), and am also a <div>.

Answer: Pink (#d53f8c) — because ID selectors have the highest specificity (0-1-0-0).

#specific-id { color: #d53f8c; } /* Specificity: 0-1-0-0 → WINS */ .specific-class { color: #dd6b20; } /* Specificity: 0-0-1-0 */ div { color: #718096; } /* Specificity: 0-0-0-1 */

6. Inheritance Demonstration

Some CSS properties are inherited from parent to child elements.

This paragraph is inside a div with color: #6b46c1 (purple). The color property is inherited, so this text is also purple.

The border property, however, is NOT inherited — only the parent div has the dashed border.

  • List items also inherit the purple color.
  • But they don't inherit the border.
.inheritance-demo { color: #6b46c1; /* INHERITED by children */ border: 2px dashed #a0aec0; /* NOT inherited */ }

7. Inherited vs Non-Inherited Properties

Inherited Properties Non-Inherited Properties
color margin
font-family padding
font-size border
font-weight background
line-height width / height
text-align display
visibility position

Use your browser's Developer Tools to inspect and debug CSS:
