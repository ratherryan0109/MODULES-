Example 3

Measure Control (max-width: 65ch)

This paragraph is constrained to 65 characters per line using max-width: 65ch. The ch unit equals the width of the "0" character in the current font, making it ideal for controlling line length. No matter how wide the viewport gets, this text will never exceed the optimal reading width.

Research has shown that 45-75 characters per line is optimal for reading speed and comprehension. Too short causes eye fatigue from frequent line breaks. Too long makes it hard to track to the next line.

Example 5

Line Height Comparison

1.2 (tight): This is a paragraph with tight line height. It works for headings but makes body text feel cramped and hard to read, especially in longer passages where tracking from line to line becomes difficult.
1.5 (normal): This is the recommended line height for body text. It provides enough vertical space between lines for comfortable reading without wasting too much vertical space. The WCAG guidelines recommend at least 1.5 for body text.
1.8 (loose): This is a generous line height often used for accessible design or poetry. While very readable, it uses more vertical space and can make scanning difficult as there's more distance between lines.
