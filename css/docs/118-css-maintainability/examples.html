CSS Maintainability

Example 1: Design Tokens as Custom Properties

All colors, spacing, and typography are defined in :root. Changing one value updates the entire system.

:root {
  --primary: #06b6d4;
  --surface: #f8fafc;
  --border: #e2e8f0;
  --radius: 8px;
  --space-sm: 0.5rem;
  --space-md: 1rem;
  --space-lg: 2rem;
}

Example 5: Consistent Component API

Buttons use the same structure: .btn block + .btn--variant modifier. No surprise overrides.

Link as Button 
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: 0.5rem 1rem;
  min-height: 44px;
  border-radius: var(--radius);
  font-size: 1rem;
  cursor: pointer;
}
.btn-primary { background: var(--primary); }
.btn-secondary { background: var(--surface); }

Example 6: Meaningful Comments

Use section comments to organize style blocks, but avoid stating the obvious.

/* ===== ABSTRACTS ===== */
:root { /* tokens */ }

/* ===== BASE ===== */
*, *::before, *::after { box-sizing: border-box; }

/* ===== COMPONENTS ===== */
.card { /* ... */ }
.btn { /* ... */ }

/* ===== LAYOUT ===== */
.l-grid { /* ... */ }

/* ===== UTILITIES ===== */
.u-flex { /* ... */ }

Example 7: Avoid Deep Nesting

Keep selector depth to 3 levels or fewer. Deep nesting creates high specificity and brittle styles.

Over-nested

.main .sidebar .widget .list .item a { }

Flat

.widget-link { } 
/* Avoid: 5 levels deep */
.main .sidebar .widget .list .item a {}

/* Prefer: one level */
.widget-link {}
