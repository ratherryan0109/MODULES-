# Mini Project: CSS Style Guide & Pattern Library

## Overview
Create a living CSS style guide and pattern library documenting all UI components, design tokens, and usage guidelines for a design system.

## Requirements
1. Document all design tokens (colors, typography, spacing)
2. Create component pages with HTML/CSS examples
3. Each component shows: default, hover, focus, active, disabled states
4. Include code snippets for each component
5. Document BEM naming conventions
6. Include usage guidelines and best practices

## Technical Requirements
- One HTML page per component
- Consistent navigation between pages
- Interactive component examples
- CSS custom properties for all tokens
- Stylelint configuration included

## Deliverables
1. `index.html` — Style guide homepage
2. `tokens.html` — Design token documentation
3. `components/button.html` — Button component page
4. `components/card.html` — Card component page
5. `components/form.html` — Form component page
6. `style.css` — Pattern library stylesheet
7. `.stylelintrc` — Stylelint config

## Evaluation
- Token documentation completeness: 20 pts
- Component state coverage: 20 pts
- Code snippet accuracy: 15 pts
- Navigation usability: 15 pts
- Naming convention consistency: 15 pts
- Usage guidelines clarity: 15 pts
