CSS Box Model - Examples

Example 1: The Four Layers of the Box Model

Every element is a rectangular box with four layers: content → padding → border → margin.

Margin (transparent space outside)
Border (visible edge)
Padding (space inside)
Content (text/images)
.element {
  margin:  1.5rem;  /* Layer 1: Outside */
  border:  3px solid #f97316;  /* Layer 2: Edge */
  padding: 1.25rem; /* Layer 3: Inside */
  /* Layer 4: Content */
}

Example 2: Box Model Calculation

Total width = content + padding + border. Margins add extra space around the box.

width: 200px
padding: 20px (each side)
border: 5px (each side)
─────────────────
Total width: 200 + 40 + 10 = 250px
This box is 250px wide total
.box {
  width: 200px;
  padding: 20px;
  border: 5px solid #06b6d4;
  /* Total width: 200 + 40 + 10 = 250px */

Example 3: Content-Box vs Border-Box

box-sizing changes how the total width is calculated.

content-box (default)
Actual: 250px
width:200px
padding:+ 40px
border:+ 10px
total:250px
border-box
Actual: 200px
width:200px (fixed)
padding:shrinks content
border:shrinks content
total:200px
.content-box { box-sizing: content-box; }  /* width = content only */
.border-box  { box-sizing: border-box; }   /* width = content + padding + border */

Example 4: Margin Collapsing

Vertical margins between adjacent elements collapse into the larger value. Horizontal margins do not collapse.

margin-bottom: 30px
margin-top: 20px

The gap between them is 30px (the larger value), not 50px.

.first  { margin-bottom: 30px; }
.second { margin-top: 20px; }
/* Collapsed margin: 30px (the larger wins) */

Example 5: Inline vs Block Box Behavior

Block boxes fill their parent width and stack vertically. Inline boxes flow horizontally and ignore width/height.

Block: full width
Block: stacks below

Inline Inline Inline (side by side) 

.block  { display: block; }   /* Full width, one per row */
.inline { display: inline; } /* Side by side, no width/height */
